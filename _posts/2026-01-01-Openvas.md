# Introduzione a OpenVAS e analisi delle vulnerabilità

---

## Introduzione

Immagina di essere il responsabile IT di un’azienda e di scoprire, quasi per caso, che un server critico è esposto a Internet con una falla di sicurezza nota. Potrebbe essere una porta aperta, un software non aggiornato o una configurazione errata: problemi che un hacker potrebbe sfruttare in pochi minuti.

Ed è qui che entra in gioco **OpenVAS**. Questo strumento open source funziona come un vero e proprio “radar per le vulnerabilità”: scansiona reti, server e applicazioni, individua i punti deboli e fornisce report chiari con consigli pratici per risolverli.

Con un database sempre aggiornato e scansioni automatizzabili, OpenVAS trasforma la complessità della sicurezza informatica in un processo gestibile, aiutando a proteggere l’infrastruttura prima che i problemi diventino emergenze.

In poche parole, OpenVAS non solo individua le falle, ma permette di agire in anticipo, migliorando la sicurezza complessiva della rete.

---

## Requisiti

- Compatibilità: Oracle VirtualBox 6.1 o superiore  
- Requisiti minimi: CPU 2 core, 5GB RAM  
- Adattatore di rete virtuale con accesso diretto a Internet  

---

## Creazione VM OpenVAS Free

1. Scaricare il file OVA dal link fornito  
2. Installare Oracle VirtualBox  
3. Avviare VirtualBox  
4. Selezionare **File > Import Appliance**  
5. Cliccare l’icona della cartella e selezionare il file OVA  
6. Cliccare **Avanti**  
7. Verificare le impostazioni della VM (modificabili)  
8. Cliccare **Import** (può richiedere fino a 10 minuti)  
9. Avviare la VM dalla barra laterale

## Setup del sistema

- Login iniziale:
  - Username: `admin`  
  - Password: `admin`  

- Wizard di configurazione:
  - Confermare con **Yes**
  - Creare utente web amministratore
  - Inserire username e password (due volte)
  - Confermare con **OK**
  - Selezionare **Skip** per feed gratuiti OpenVAS  

---

## Accesso interfaccia web

1. Aprire il browser  
2. Inserire l’indirizzo IP della VM  
3. Effettuare login con le credenziali create  

---

## Note aggiuntive

- Cambiare subito la password di amministratore (fondamentale per la sicurezza)  
- OpenVAS Free usa un certificato TLS self-signed → aggiungere eccezione nel browser  
- Spegnere la VM solo da: **Maintenance > Power**

# Risultati delle scansioni

---

## Windows 7

### Problemi principali

- Sistema operativo obsoleto → nessun supporto e patch  
- Vulnerabilità critiche:
  - EternalBlue (SMBv1 / MS17-010) → esecuzione remota di codice e ransomware (es. WannaCry)  
  - Servizi RPC esposti (porta 135) → raccolta informazioni  
  - TCP Timestamps → fingerprinting e analisi uptime  

### Cosa fare

- Migrare a Windows 11 o Linux supportato  

### Mitigazioni temporanee

- Isolamento in VLAN dedicata  
- Blocco accesso Internet diretto  
- Firewall e IDS/IPS  
- Backup frequenti  

---

## Windows XP

### Problemi principali

1. Sistema operativo EOL  
   - Nessun aggiornamento dal 2014  
   - Rischio massimo (CVSS 10.0)  

2. Vulnerabilità SMB (MS09-001)  
   - Porta 445  
   - Esecuzione remota senza autenticazione  

3. EternalBlue (MS17-010)  
   - Worm propagation (WannaCry)  
   - RCE via SMBv1  

### Cosa fare

- Migrazione immediata a sistemi moderni  
- Disabilitare SMBv1  
- Bloccare porta 445  
- Segmentazione rete  
- Monitoraggio continuo  

---

## Windows 10

### Problemi principali

- Porta 445 (SMB)
  - Credenziali di default presenti  
  - Accesso non autorizzato e furto dati  

- Porta 135 (DCE/RPC)
  - Esposizione informazioni di sistema  

### Cosa fare

- Cambiare credenziali di default  
- Bloccare porte 445 e 135  
- Aggiornamenti costanti  
- Disabilitare servizi non necessari  

---

## Ubuntu 18

### Problema principale

ICMP Timestamp Reply (CVSS 2.1)

- Esposizione timestamp interni:
  - ricezione  
  - trasmissione  

### Impatto

- Fingerprinting del sistema  
- Nessun impatto diretto su dati o esecuzione codice  

### Mitigazione

- Disabilitare ICMP Timestamp  
- Filtrare ICMP via firewall  
- Hardening rete  

---

## Metasploitable 2

### Problemi principali

1. OS obsoleto (Ubuntu 8.04 EOL)  
2. Backdoor vsftpd 2.3.4 (porta 6200)  
3. Servizi legacy insicuri (rlogin, rsh, rexec)  
4. Configurazioni pericolose:
   - HTTP PUT/DELETE  
   - FTP credenziali default  
   - Telnet in chiaro  
5. Vulnerabilità diffuse:
   - Samba RCE  
   - phpinfo exposure  
   - directory listing  

### Cosa fare

- Eliminare o aggiornare sistema  
- Rimuovere servizi compromessi  
- Disabilitare servizi legacy  
- Hardening completo  
- Vulnerability assessment regolari  

---

## Ubuntu 22 + OwnCloud

### Problema principale

- HTTP senza TLS (CVSS 4.8)  
- Basic Authentication in chiaro  

### Rischi

- Intercettazione credenziali (MITM)  
- Accesso non autorizzato  
- Compromissione account  

### Mitigazioni

- Abilitare HTTPS  
- Redirect HTTP → HTTPS  
- Disabilitare autenticazione non cifrata
