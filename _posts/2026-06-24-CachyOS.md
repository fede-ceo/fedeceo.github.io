# Sfida tra Scheduler: CachyOS vs Ubuntu nella Compilazione del Kernel Linux

Prima di analizzare i dati, è essenziale capire cos'è uno scheduler. Nei sistemi operativi, lo scheduler è il componente del kernel che decide quale processo deve accedere alla CPU e per quanto tempo. Il suo compito è massimizzare l'efficienza, bilanciando la reattività del sistema con la potenza di elaborazione, specialmente nei carichi multi-core.

In questo articolo misuriamo oggettivamente come le scelte di scheduling influenzano un'operazione pesante: la compilazione del Kernel Linux.

## CachyOS vs Ubuntu

* **Ubuntu:** Utilizza un kernel generico pensato per la massima compatibilità, basandosi sugli scheduler standard di Linux.
* **CachyOS:** Punta alle prestazioni pure. Sfrutta set di istruzioni moderne (x86-64-v3/v4) e kernel personalizzati con scheduler avanzati per ridurre la latenza e ottimizzare il throughput.

## L'Ambiente di Test

Ho configurato due macchine virtuali identiche su VirtualBox:

* **RAM:** 8 GB
* **CPU:** 4 Core
* **Target:** Kernel Linux 7.1

## La Procedura

### 1. Prerequisiti

Su Ubuntu:
```bash
sudo apt update && sudo apt install build-essential libncurses-dev bison flex libssl-dev libelf-dev bc wget
```

Su CachyOS:
```bash
sudo pacman -Syu base-devel bc ncurses bison flex openssl libelf wget
```

### 2. Download ed estrazione
```bash
wget [https://cdn.kernel.org/pub/linux/kernel/v7.x/linux-7.1.tar.xz](https://cdn.kernel.org/pub/linux/kernel/v7.x/linux-7.1.tar.xz)
tar -xf linux-7.1.tar.xz
cd linux-7.1
```

### 3. Configurazione e Avvio

Configurazione base per un carico identico:
```bash
make defconfig
```

Compilazione misurata su tutti i core:
```bash
time make -j$(nproc)
```

## I Risultati

* **CachyOS:** Real 6m 39s | User ~17m | Sys ~5m
* **Ubuntu:** Real 7m 39s | User ~18m | Sys ~6.5m

### 💡 Analisi dei dati e Metriche

CachyOS chiude l'operazione con 1 minuto esatto di vantaggio. Per capire a fondo questo divario, è fondamentale analizzare i parametri registrati dal comando `time`:

* **Real Time:** È il tempo reale misurato dall'orologio, ovvero i minuti effettivi trascorsi dall'avvio del comando al suo termine.
* **User Time:** Rappresenta il tempo totale di CPU impiegato dal codice in "user space" (spazio utente). Essendo un'operazione parallelizzata su 4 core, questo valore è la somma matematica del lavoro svolto simultaneamente da ciascun core.
* **Sys Time:** È il tempo totale impiegato dal kernel per gestire le chiamate di sistema necessarie al processo, come l'I/O sui file e l'allocazione della memoria.

I dati confermano la maggiore efficienza di CachyOS. Il minor Sys time (5m contro 6.5m) indica che il kernel di CachyOS gestisce le risorse e le comunicazioni con l'hardware in modo nettamente più snello. Questo dimostra che le ottimizzazioni architetturali e uno scheduler più aggressivo garantiscono prestazioni superiori in scenari di calcolo intensivo.
